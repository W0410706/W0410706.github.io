Gap Analysis: Vue.js

What is Vue.js?

Vue.js is a progressive framework for JavaScript, similar to Angular or React, used for building user interfaces and front-end applications.

Why use it?

Vue.js is approachable, versatile, performant, maintainable, and testable. It can easily be integrated with other libraries or existing projects. It is also easy to learn and useful for building lightweight, powerful Single Page Applications. Vue.js is rapidly gaining in popularity, so it could become important for front-end web developers to know.

Vue Components

Vue components are reusable components that hold data and can pass data to other components through props (properties held within a component tag). Each Vue component is a single file ending in .vue, with App.vue as the root component that contains the other components of the application and the global styles. Components can be nested within one another so that there are several layers between App.vue and the innermost component.

Component Anatomy

Every component contains the code that defines its structure, logic, and appearance:

  • < template > contains the HTML for output/structure.
  • < script > contains the JavaScript for functionality/logic.
  • < style > contains the CSS for appearance/style.

Vue Templating ----------------------------->

Vue Interpolation

Interpolation, denoted with two sets of curly braces, is used within Vue Templating in order to display something to the screen. It cannot be applied to attributes; it must be used between html tags. JavaScript variables or expressions (like ternary operators) can be put within interpolations.

Vue Directives

Vue Directives are denoted by v- and are also used in Vue Templating. Vue directives are meant to be used within an html element’s opening tag to do a variety of things. They can be triggers for functions on certain events or can make conditional calls to functions and can be used for many other purposes. There are about a dozen Vue directives, but some of the most common are the following:

  • v-for
  • v-if
  • v-bind
  • v-on

v-on Shorthand

The @ symbol can be used as a shorthand for v-on. It is completely optional to use, but it is there to simplify code.

Some common uses for v-on shorthand:

  • @click
  • @submit
  • @[event]

Class & Style Binding

Can bind a class to an html element to then target with the css, so styles for this class can be defined in the < style > section of the component. This is especially nice for attaching a class on the fly depending on whether an event is triggered.

The code for binding a class looks like this:
v-bind:class=“ … ”

It is also possible to bind inline styles to an html element. The code for this looks like:
v-bind:style=“ … ”
